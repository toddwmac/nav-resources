💡 Understanding the Data Flow

🔄 Master Data Loading

💾 Local Edit Storage

Any changes you make are saved to your browser's localStorage. Your edits persist across sessions but don't modify the master file.

ℹ️ About This Collection

🎯 Purpose: AI Model Showcase

This collection demonstrates how different AI models (Claude, Copilot, GPT) approach the same application requirements. Each Nav Resource Manager provides full CRUD functionality but showcases unique design patterns, code organization, and UI/UX choices made by different AI systems.

New: A mobile-optimized app with change tracking lets you export edits for merging into the master database, enabling consistent updates across all apps.

📊 Master Database System

Initial Load: All applications automatically load from master-resources.json containing 95+ curated navigation resources on startup.

Local Editing: Any changes you make (add, edit, delete) are saved to your browser's localStorage and persist across sessions. These edits are local only and don't modify the master file.

Change Tracking (Mobile): The mobile app tracks all changes and can export them as JSON. Exported changes can be merged into the master database to update all apps consistently.

Data Flow: Master File → App Initialization → Local Storage (your edits) → Display

⚙️ How It Works

Master Database

💾

Your edits (add, modify, delete) are saved to browser localStorage automatically. Changes persist across sessions but remain local to your browser only.
